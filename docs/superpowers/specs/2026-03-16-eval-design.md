# 离线 Eval 设计

**目标**

为本地券商研报 RAG 项目增加一个可重复运行的离线评测能力，直接走真实检索链路和真实本地模型，输出检索层与回答层的质量指标。

**范围**

- 新增从现有项目数据自动生成评测样本的能力
- 新增离线评测入口，直接调用现有 `ResearchRAGPipeline`
- 输出检索命中、答案要点覆盖和拒答准确率
- 不引入额外评审模型，不修改线上 API

**设计**

- 在 `evals/` 下新增数据集生成器与评测脚本。
- 数据集生成器扫描 `data/projects/*/parsed_markdown/**/*.md`，从标题清晰、信息密度较高的段落中抽取候选事实，自动生成正样本与拒答样本。
- 每条样本包含 `project_name`、`question`、`expected_reports`、`expected_section_keywords`、`expected_answer_points`、`should_refuse`、`source_type` 等字段。
- 评测脚本直接实例化现有 `ProjectManager` 与 `ResearchRAGPipeline`，对每条样本运行真实 `answer_question()`。
- 检索层依据返回的 `sources` 判断是否命中目标报告与目标章节关键词；回答层使用规则评分，统计答案要点覆盖率和拒答正确率。
- 评测结果写入 `evals/results/<timestamp>.json`，同时输出终端摘要。

**数据生成策略**

- 正样本仅从可定位章节的高信息段落生成，优先保留数字、同比环比、评级、结论性表述。
- 初版跳过复杂表格，避免自动生成不可验证的样本。
- 负样本通过错主体、错指标、跨项目问题等方式构造，确保当前项目证据不足。
- 同一文档限制样本数量，避免数据集分布过于集中。

**错误处理**

- 当项目下没有可用 Markdown 时，生成器跳过该项目并给出统计。
- 当真实链路调用失败时，评测脚本记录单条错误，不中断整个批次。
- 当答案为空或检索为空时，仍输出完整的单样本结果，方便定位失败环节。

**验证**

- 为数据集生成、规则评分和结果汇总增加单元测试。
- 使用仓库内真实数据生成一份样本集，再执行一次评测脚本的最小烟雾验证。
