# 项目删除功能设计

**目标**

为前端项目列表中的每个项目提供一个删除按钮；用户确认后，删除整个项目及其全部派生产物，包括 PDF、向量库和解析产物。

**范围**

- 前端项目列表新增删除按钮
- 删除前弹出确认框
- 后端新增删除整个项目的 API
- 删除项目根目录下的全部内容

**设计**

- 后端在 `backend/project_manager.py` 中新增项目删除方法，直接删除项目根目录。
- API 在 `app.py` 中新增 `DELETE /api/projects/{project_name}`。
- 前端在 `frontend/app.js` 的项目行中新增删除按钮，样式复用现有文档删除按钮风格。
- 删除成功后刷新项目列表；若当前激活项目已被删除，则切换到剩余第一个项目，若无项目则清空当前状态。

**交互**

- 删除前使用浏览器原生 `window.confirm` 二次确认。
- 确认文案明确说明会删除 PDF、向量库和解析产物。
- 删除中按钮禁用，防止重复提交。

**错误处理**

- 项目不存在时返回 404。
- 项目名非法时返回 400。
- 文件系统删除失败时返回 500，并在前端提示失败原因。

**验证**

- 仓库当前没有现成测试目录，本次不新增测试框架。
- 通过 Python 语法编译校验后端改动，通过 Node 语法检查校验前端脚本。
